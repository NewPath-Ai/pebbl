# Eval: Google OKF vs Pebbl - what's worth stealing

> **Historical record.** QMD (the `@tobilu/qmd` semantic search index) was removed from pebbl in M2. SQLite FTS5/BM25 is now the only search engine. This note predates that change and is kept as a design/decision record; its QMD references describe how pebbl worked at the time, not current behavior.

*Verdict: don't adopt OKF in any evaluated form. Adversarially verified (16-agent workflow, 5 of 6 claims killed). The investigation's real yield is confirming one already-known wound, not an OKF feature. June 17, 2026.*

Triggered by Google's Open Knowledge Format announcement (v0.1, ~2026-06-12). The question was whether OKF should change Pebbl, or improve the factory.

---

## TL;DR

- **Steal nothing from OKF as a format.** It's a producer-only ecosystem today: every shipped tool writes OKF, the only confirmed reader is Google's own Knowledge Catalog. Interop value is near-zero now. Same call we made on [Gantry] and [Beads] - steal ideas, not the thing - except here even the ideas mostly don't survive scrutiny.
- **Keep SQLite as the source of truth.** The one claim that survived adversarial review. Don't flip to markdown-truth.
- **The one real wound the comparison surfaced is already ours:** foundation/component "why" is trapped in gitignored `db.sqlite`, so a fresh clone gets code and no reasoning. That's [design-shared-decisions.md](./design-shared-decisions.md) Option C, not an OKF feature. OKF contributes at most a near-free `type:` frontmatter hedge if/when that ships.
- **OKF does not help the factory carry-gap.** That's an enforcement problem, not a packaging one.

## What OKF is

A directory of markdown files with YAML frontmatter, distributed via git. Exactly one required field (`type`); recommended `title`/`description`/`resource`/`tags`/`timestamp`. Links between files form an untyped knowledge graph. Reserved `index.md` (progressive disclosure) and `log.md` (dated history). Conformance bar (SPEC §9) is trivial: parseable frontmatter + non-empty `type` per file. Most meaning lives in prose and is not machine-checkable.

## The architecture finding (why Pebbl already wins)

The first instinct - "OKF is basically Pebbl" - is wrong. Pebbl's source of truth is **SQLite**; the markdown files are regenerable, gitignored projections that exist only because qmd indexes markdown and humans read it (PEBBL.md:69-83). OKF's whole bet is that the markdown **is** the truth.

Once you see that, Pebbl out-designs OKF on nearly every axis OKF cares about:

| OKF feature | Pebbl equivalent | Winner |
|---|---|---|
| static `index.md` | computed `pebbl context` (narrative + topics + recent) | Pebbl |
| `log.md` dated history | timestamps + commit-log + archive + `--corrects` chains | Pebbl |
| untyped markdown links | typed integer FKs (`relates_to`, `corrects`) | Pebbl |
| permissive consumption | rubric falls back to `uncategorized` | tie |
| retrieval | qmd semantic + keyword fallback | Pebbl (OKF has none) |
| classification / decay | rubric + tiers + atomic compaction | Pebbl (OKF has none) |

OKF wins on exactly one axis - a portable, human-readable, git-diffable source of truth - and that's the axis Pebbl deliberately traded away for atomic compaction and addressable correction chains.

## Adversarial verdicts

Six load-bearing claims, each handed to a skeptic told to refute (default to killed).

| # | Claim | Verdict | Conf |
|---|---|---|---|
| 1 | Add `pebbl export` emitting an OKF bundle, worth the effort | **killed** | med |
| 2 | Replace HTML-comment metadata with YAML frontmatter (`type`) | **killed** | high |
| 3 | Add OKF `resource:` field pointing a note to the file/line it describes | **killed** | high |
| 4 | OKF meaningfully helps the factory intent carry-gap | **killed** | high |
| 5 | Pebbl should NOT flip source of truth to markdown right now | **survives** | high |
| 6 | OKF portability has real value today | **killed** | high |

Why each killed claim failed:

- **(1)** The value people reach for is the ADR export (Option C), which is a generic git-trackable-markdown win. OKF conformance (one `type` field) is a thin coat of paint that adds no machine-checkable meaning and optimizes for a reader market that doesn't exist.
- **(2)** The HTML-comment metadata (`<!-- cat:X topic:Y tier:Z -->`) is a *live machine-parse contract* produced in 5+ sites and consumed by search/mirror/context/narrative. Replacing it in place is a coordinated producer+consumer rewrite with real regression risk, for near-zero readership. Fine only as a *new export target*, never an in-place swap.
- **(3)** Wrong layer: `pebbl check` reads `db.sqlite` (check.js:90), not the projection - an OKF frontmatter field never reaches it. OKF `resource` is a whole-asset URI and can't encode a line. And the capability already ships: `check.js:25-71` regex-scrapes cited paths and `fs.existsSync`-validates them. The real improvement is a structured `cites` *column* on the logs table, not an OKF key.
- **(4)** The carry-gap is enforcement, not packaging. `agent.sh:90-118` already forces the builder to open/restate/append the traveler workpad, but `review-stage-a.sh` (the zero-LLM gate) receives the traveler path and never checks it - read-first is honor-system. A format defines no read-and-acknowledge obligation; a builder ignores an OKF bundle as easily as it lies about reading the traveler. Fix is a mechanical assertion in stage A.
- **(6)** Every OKF tool is a producer; the only reader is Google's Knowledge Catalog. The genuine present-day benefit ("commit a markdown projection that travels with a clone") is a plain-markdown win, not an OKF-portability win.

## The one real wound (already designed, not OKF)

Foundation/component decisions live only in local gitignored `db.sqlite`. A fresh clone - a new contributor, or the factory building pebbl - gets code and none of the why. [design-shared-decisions.md](./design-shared-decisions.md) already solves this: **Option C**, a privacy-gated `pebbl export-decisions` writing a tracked, deterministic `DECISIONS.md` of the top two tiers, with `--check` freshness and struck-through supersession rendering.

If that ships, OKF frontmatter on the output is a near-free hedge against a future reader market - emit it, don't build *for* it. Three corrections the OKF export sketch got wrong and any build must respect:

1. **Privacy gates FIRST, not as a fast-follow.** The spike is emphatic: entries are excluded by default, shared only when explicitly marked, "because a leak is the one failure that can't be walked back." A naive tracer that `SELECT`s all foundation+component rows into a tracked dir leaks by default. Don't.
2. **The `narrative.js` "stamp" to copy doesn't exist.** narrative.js writes `<!-- updated -->`/`<!-- refs -->`, not `<!-- exported-from: db@<max-id> -->`. The freshness watermark is net-new code (both the spike and the export sketch assume otherwise).
3. **Layout is an open fork, not settled.** The spike chose a single grouped `DECISIONS.md`; one-file-per-entry (better link addressability) re-introduces the ungrouped weakness the spike rejected for human ADR reading. Decide deliberately.

Also reconfirmed (separately tracked in the audit): README says "commit `.pebbl/`" while `init.js` gitignores it - a real contradiction for consuming projects.

## Factory

`okf-irrelevant`. OKF appears nowhere in the factory (grep clean), and the factory already rejected a separate AI-maintained store on DRY grounds (2026-06-17). The carry-gap's real fix is mechanical: have `review-stage-a.sh` assert the active traveler gained a fresh timestamped Progress entry + goal restatement, and hard-fail if absent (review.sh already passes it `$TASKFILE`). That converts read-first from honor-system to enforced. Process change, not a format. Tracked as slice S1 hardening, out of scope for this eval.

## Decision

Do not adopt OKF. Keep SQLite as truth. The genuinely worth-building item the investigation surfaced is Option C (shareable decisions), which stands on its own merit as plain markdown and predates OKF. Treat OKF frontmatter as an optional, near-zero-cost garnish on that export if and when it ships, gated behind privacy. Revisit OKF only if a real independent *consumer* ecosystem appears (>= 1.0 + a reader that isn't Google).
